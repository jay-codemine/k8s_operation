# 📄 k8soperation 前端管理系统部署文档

本文档用于说明 **k8soperation 前端管理系统（k8s-web）** 的部署方式，  
适用于 **开发环境 / 测试环境 / 生产环境**。

---

## 一、前端项目说明

- 项目目录：`k8s-web/`
- 技术栈：Vue 3 + Vite + Arco Design
- 构建产物：静态资源文件
- 部署方式：Nginx / Docker / Kubernetes

---

## 二、前端构建

### 1. 安装依赖

```bash
cd k8s-web
npm install
```



### 2. 配置环境变量

编辑 `.env` 文件：

```js
VITE_API_BASE=http://<backend-host>:8080/api/v1
```

### 3. 构建前端静态文件

```bash
npm run build
```

构建完成后，会生成目录：

```bash
dist/
```

------

## 三、Nginx 部署（推荐）

### 1. Nginx 配置示例

```bash
server {
    listen 80;
    server_name k8soperation.example.com;

    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 2. 拷贝构建产物

```bash
cp -r dist/* /usr/share/nginx/html/
```

------

## 四、Docker 部署

### 1. Dockerfile 示例

```dockerfile
FROM nginx:alpine
COPY dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
```

### 2. 构建与运行

```dockerfile
docker build -t k8soperation-web .
docker run -d -p 80:80 k8soperation-web
```

------

## 五、Kubernetes 部署（可选）

- 使用 Deployment + Service
- 通过 ConfigMap 管理 `nginx.conf`
- 使用 Ingress 暴露访问入口

> 详细示例可参考后端部署文档中的 Ingress 配置方式。

------

## 六、常见问题

### 1. 刷新页面 404

请确保 Nginx 中配置：

```bash
try_files $uri $uri/ /index.html;
```

### 2. 接口跨域问题

推荐通过 Nginx 反向代理 `/api/`，避免 CORS 问题。

------

## 七、与后端对接说明

- 所有接口统一使用 `/api/v1` 前缀
- 使用 JWT + Refresh Token 认证
- 未授权请求返回 401，并由前端自动跳转登录页

## 🔗 API 接口规范说明

本系统所有 API 统一使用 `/api/v1` 作为版本前缀，  
按业务域拆分为认证接口与 Kubernetes 业务接口。



### 认证接口（Auth）

```bash
/api/v1/auth/*
```

用于处理用户身份相关操作，如登录、注册、Token 刷新等，
该类接口通常不需要携带 JWT。

### Kubernetes 业务接口（K8s）

```bash
/api/v1/k8s/*
```

用于处理 Kubernetes 资源管理操作，
所有接口均需要通过 JWT 鉴权，并支持多集群访问。

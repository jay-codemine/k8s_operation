## 1. Deployment 滚动更新策略

### 当前实现状态

**代码位置**: `pkg/k8s/deployment/builders.go`

#### 问题发现:
- ❌ **未显式设置 UpdateStrategy**
- 当前依赖 Kubernetes 默认行为（RollingUpdate）
- 缺少 `maxSurge` 和 `maxUnavailable` 参数控制

#### 默认行为:
```go
// 当前代码（第180-189行）

    ObjectMeta: dpMeta,
    Spec: appv1.DeploymentSpec{
        Replicas:             &replicas,
        Selector:             &metav1.LabelSelector{MatchLabels: selector},

        RevisionHistoryLimit: &revisionHistoryLimit,
        // ⚠️ 未设置 Strategy 字段
    },

```

#### 建议补充:
```go

    ObjectMeta: dpMeta,
    Spec: appv1.DeploymentSpec{
        Replicas:             &replicas,
        Selector:             &metav1.LabelSelector{MatchLabels: selector},
        Template:             podTemplate,

            Type: appv1.RollingUpdateDeploymentStrategyType,
```

```go
func PatchImageStatefulSet(ctx context.Context, Kube kubernetes.Interface,

    if err != nil {
```

## 附录：Kubernetes 滚动更新策略说明

### Deployment RollingUpdate 参数

- **maxSurge**: 滚动更新时最多可以超出期望副本数的 Pod 数量（默认 25%）
- **maxUnavailable**: 滚动更新时最多允许不可用的 Pod 数量（默认 25%）

### StatefulSet RollingUpdate 参数

- **partition**: 指定从哪个序号开始更新（例如 partition=2，则只更新 pod-2、pod-3...）
- 更新顺序：从最大序号到最小序号（逆序）

### DaemonSet RollingUpdate 参数

- **maxUnavailable**: 滚动更新时最多允许不可用的 Pod 数量（默认 1）
- 更新策略：逐个节点更新，等待 Pod Ready 后再更新下一个节点
